# 测试计划 (TEST_PLAN)

> 依据实施计划书 §20、§21。

## 一、后端自动化测试（pytest + TestClient）

### health
- health 正常返回 ok

### scan
- 新标签扫描返回 `new`
- 已知标签扫描返回 `known`
- 重复 UID 绑定返回 409

### toys
- 创建玩具输入校验（名字为空、过长）
- 查询玩具
- 编辑玩具

### play
- 开始会话返回 session_id + wake_phrase + spark
- 下一条火花
- 已结束会话不能继续（返回 409）
- 每会话火花上限

### memories
- 保存记忆
- 删除记忆
- 第二次开始时读取记忆

### AI 降级
- AI 超时降级
- AI 非 JSON 降级
- AI 长文本降级
- 固定内容不重复（同会话内）

## 二、Android 自动化测试（单元测试，纯逻辑）

- Tag UID 格式化（大写十六进制、无分隔）
- NFC 去重逻辑（500-1000ms 窗口）
- API DTO 映射
- ViewModel 状态流转

## 三、手工验收用例

### 用例 A：新玩具
1. 使用未绑定标签扫描 → 进入绑定
2. 名字留空，确认无法提交
3. 填写"大狮子"保存 → 开始游戏
4. 预期：数据保存，语音正常

### 用例 B：已知玩具
1. 返回扫描页再扫大狮子 → 自动识别
2. 播放唤醒语 + 游戏火花
3. 预期：无需再次绑定

### 用例 C：AI 失败
1. 将 AI Key 改错 → 扫描大狮子开始游戏
2. 预期：固定内容正常播放，App 不报技术错误

### 用例 D：保存记忆
1. 输入"孩子让大狮子坐消防车" → 结束游戏 → 保存记忆
2. 第二天重新开始
3. 预期：后端存在记忆，AI 可引用最多一条

### 用例 E：重复 NFC
1. 手机持续贴着标签观察 5 秒
2. 预期：只创建一次扫描处理和一个会话
